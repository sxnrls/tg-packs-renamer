# 🎭 Tg packs renamer

> Массовое переименование стикерпаков и эмодзи-паков в Telegram — через один запуск скрипта.

Если у вас **сотни стикерпаков** с рекламным `@username` в названии и вы хотите сменить его везде разом — этот инструмент для вас. Никаких ручных правок. Никаких ботов. Прямой доступ через MTProto.

---

## ✨ Что умеет

- 🔄 Меняет `@username` в **названии** пака (не ссылку на сам пак)
- 📦 Работает со **стикерпаками**, **эмодзи-паками** или с обоими сразу
- 🚀 Обходит **все ваши паки** которые вы создали
- 🛡️ Уважает **FloodWait** — ждёт и повторяет попытку, не роняет сессию
- ❌ Поддерживает **исключения** — можно пропустить конкретные паки
- 💾 Сохраняет API-данные в `.env` — при следующем запуске вводить не нужно
- 🎨 Красивый интерактивный интерфейс прямо в терминале

---

## 📋 Как это работает

**До:**
```
один @username          →   один @your_new_username
два@user       →   два @your_new_username   (добавлен пробел!)
три                   →   три @your_new_username   (@ добавлен в конец)
```

**После:** всё переименовано, `@username` кликабелен.

## ⚙️ Настройки задержки

В коде можно изменить параметр `delay` в функции `rename_all()`:

```python
await rename_all(client, params, delay=2.5)
```

| Значение | Скорость | Безопасность |
|----------|----------|--------------|
| `1.0` | Быстро | Риск FloodWait |
| `2.5` | Умеренно | ✅ Рекомендуется |
| `5.0` | Медленно | Максимальная безопасность |

> При 2.5 секунды ~700 паков переименовываются примерно за **30 минут**.

## ❓ FAQ

**Меняется ли ссылка на сам пак?**
Нет. Ссылка `t.me/addstickers/short_name` не меняется — только отображаемое **название** пака.

**Можно ли потом переименовать снова с другим @?**
Да. Просто запустите скрипт ещё раз и укажите новый username.

**Что если у пака нет @ в названии?**
Скрипт добавит ` @username` в конец названия.

**Безопасно ли это для аккаунта?**
Скрипт работает через официальный MTProto API, соблюдает FloodWait и делает паузы между запросами. При настройках по умолчанию риск ограничений минимален.
